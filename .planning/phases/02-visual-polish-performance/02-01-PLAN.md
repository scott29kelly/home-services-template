    Place inside `<BrowserRouter>` but outside `<App />`.

    **Step 2: Migrate all motion.* to m.* (18 files)**

    In every file listed above:
    1. Change `import { motion } from 'framer-motion'` to `import { m } from 'framer-motion'`
    2. If file also imports other things like `AnimatePresence`, `useScroll`, `useTransform`, `useInView` -- keep those as-is, only change `motion` to `m`
    3. Replace ALL `motion.div`, `motion.h2`, `motion.p`, `motion.span`, `motion.section`, `motion.li`, etc. with `m.div`, `m.h2`, `m.p`, `m.span`, `m.section`, `m.li`, etc.
    4. Do NOT change `AnimatePresence`, `useScroll`, `useTransform`, `useInView` -- these are standalone imports unaffected by LazyMotion

    **Step 3: Fix hero above-fold animation (Hero.tsx)**

    The hero text currently has `initial={{ opacity: 0, y: 40 }}` which delays LCP by ~800ms. Remove the wrapping `motion.div` (now `m.div`) around the hero text content entirely -- make it a plain `<div>`. The hero text, subtitle, and CTA buttons MUST render instantly with no animation delay.

    The hero background image can keep a subtle CSS treatment (e.g., a slight scale-up via CSS animation or a gentle parallax), but the TEXT and BUTTONS must be visible at first paint.

    **Step 4: Remove layout prop from Projects page (Projects.tsx)**

    The Projects page uses `layout` prop on `motion.div` (now `m.div`) for filter transitions. The `layout` prop requires `domMax` (+25KB) which we are NOT loading. Remove ALL `layout` props from this file. Instead, rely on `AnimatePresence` with `mode="popLayout"` for enter/exit transitions when filtering, and CSS transitions for any remaining smooth movements. The filter animation should still feel smooth -- just not using layout animations.

    **Step 5: Verify all scroll animations use subtle fade-up per user decision**

    Per CONTEXT.md locked decision: scroll animations should be "subtle fade-up only (~20px slide + opacity)". As you migrate each component, ensure animation values use approximately `initial={{ opacity: 0, y: 20 }}` (not y: 40 or larger values). Adjust any that use larger y values down to ~20px. This applies to all scroll-triggered section animations (BentoGrid, Stats, ProcessTimeline, Testimonials, Certifications, CTA, FAQ, etc.).

    Do NOT change the easing curves -- keep existing `ease: [0.25, 0.1, 0.25, 1]` pattern.

    ```typescript
    interface SectionHeadingProps {
      title: string
      subtitle?: string
      align?: 'center' | 'left'   // default: 'center'
      as?: 'h1' | 'h2' | 'h3'    // default: 'h2'
      theme?: 'dark' | 'light'    // dark = white text on dark bg, light = navy text on light bg. Default: 'light'
      animated?: boolean           // default: true (uses m component for fade-up)
      className?: string           // optional extra classes on the wrapper div
    }
    ```

    Implementation:
    - Use `m` component from framer-motion for the fade-up animation (subtle: `initial={{ opacity: 0, y: 20 }}`, `whileInView={{ opacity: 1, y: 0 }}`, `viewport={{ once: true, amount: 0.5 }}`)

    - Subtitle: `text-lg max-w-2xl` with color based on theme (dark: `text-white/80`, light: `text-slate-600`) and `mx-auto` when centered
