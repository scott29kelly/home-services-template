    **Step 3: Add width/height attributes to ALL img tags**

    Audit every component that renders `<img>` tags and add explicit `width` and `height` attributes. This prevents CLS (Cumulative Layout Shift).

    For responsive images, the aspect ratio matters more than exact pixels. Use the natural dimensions of the source images, or use representative dimensions that maintain the correct aspect ratio.

    Common image sizes in this project:
    - Hero images: `width="1920" height="1080"` (16:9)
    - Project gallery: `width="800" height="600"` (4:3)
    - Testimonial images: `width="600" height="400"` (3:2) or similar
    - Team photos: `width="400" height="400"` (1:1) or `width="400" height="500"` (4:5)
    - Logo/certification images: varies

    Search for all `<img` tags across src/ and add width/height. If the image is inside a container that controls its size via CSS (e.g., `className="w-full h-48 object-cover"`), still add the width/height for CLS prevention -- the CSS will override the rendered size but the browser uses the attributes for aspect ratio calculation.

    **Step 4: Set loading attributes on all images**

    - **Above-fold images (Hero):** Add `fetchPriority="high"` and do NOT add `loading="lazy"`. The hero image should load eagerly.
    - **Below-fold images (everything else):** Add `loading="lazy"` and `decoding="async"` to every `<img>` tag that is NOT in the hero/above-fold.

    Files to check and update:

    - `Projects.tsx` -- project images: lazy load
    - `About.tsx` -- team/about images: lazy load
    - `ServicePage.tsx` -- service page images: lazy load
    - Any other components with `<img>` tags
