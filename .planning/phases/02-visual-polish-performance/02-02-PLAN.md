    **Step 1: Build BeforeAfterSlider component**

    Create `src/components/ui/BeforeAfterSlider.tsx` using the Pointer Events API for unified mouse/touch handling.

    Props:
    ```typescript
    interface BeforeAfterSliderProps {
      beforeImage: string
      afterImage: string
      beforeLabel?: string   // default: 'Before'
      afterLabel?: string    // default: 'After'
      className?: string
    }
    ```

    Implementation details (per research Pattern 3):
    - Use `onPointerDown`, `onPointerMove`, `onPointerUp` on the container for unified mouse/touch input
    - Call `setPointerCapture(e.pointerId)` on pointer down so drag continues even if pointer leaves element
    - Set `touch-action: none` on container to prevent default touch scroll
    - State: `position` (0-100 percentage) starting at 50
    - After image: full-width background
    - Before image: overlaid with `clipPath: inset(0 ${100 - position}% 0 0)` to clip from right
    - Draggable divider: vertical line at `left: ${position}%` with a handle grip (circle with arrows icon or grip lines)
    - Handle styling: white circle with shadow, contains a grip indicator (two small chevron arrows pointing left/right, or a grip icon from lucide-react). Should be visually prominent and obviously draggable.
    - Add "Before" and "After" labels in the top corners of each respective side
    - **ARIA accessibility:** `role="slider"`, `aria-valuenow={Math.round(position)}`, `aria-valuemin={0}`, `aria-valuemax={100}`, `aria-label="Before and after comparison slider"` on the handle
    - **Keyboard:** `onKeyDown` handler on the handle -- ArrowLeft/ArrowRight adjust position by 5%, Home sets 0%, End sets 100%
    - **Mobile:** The component should work well at full-width on mobile. Touch drag should feel smooth.
    - Handle `tabIndex={0}` for keyboard focus
    - Both images should have matching `width` and `height` to prevent CLS
