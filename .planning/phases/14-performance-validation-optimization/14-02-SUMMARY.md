## Issues Encountered

**Duplicate hero preload in rendered HTML:** The built index.html contains two `rel="preload" as="image"` links — one auto-generated by React Router's SSR from the Hero component's `<img fetchPriority="high" srcSet>`, and one from our explicit preload in root.tsx. Both now share the same `imageSrcSet` values. Modern browsers deduplicate matching preloads, so this is not a functional issue. The plan anticipated this behavior: "the browser deduplicates matching preloads."

**Mobile performance does not improve to 90+:** This was anticipated in the plan. The optimizations (async fonts + responsive preload) improved scores by 0-3 points, but the fundamental bottleneck is Lighthouse's simulated mobile throttling (4x CPU, 3G network) combined with the React hydration cost and Google Fonts external origin. Reaching 90+ requires self-hosting fonts and significant JS bundle reduction — architectural changes deferred per user decision.
