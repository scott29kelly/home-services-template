  <action>
Convert the simple route wrapper files that use `import PageComponent from '../pages/PageComponent'` to use `React.lazy()` with `Suspense`. This applies to route files that are thin wrappers around page components (NOT route files with inline loaders/logic like `service-page.tsx`, `service-areas.$slug.tsx`, `portfolio.$slug.tsx`, `resources.$slug.tsx` — leave those unchanged since their page logic is inline).

**Pattern for each wrapper route:**

Before (e.g., `routes/about.tsx`):
```tsx
import About from '../pages/About'

export default function AboutRoute() {
  return <About />
}
```

After:
```tsx
import { lazy, Suspense } from 'react'

const About = lazy(() => import('../pages/About'))

export default function AboutRoute() {
  return (
    <Suspense fallback={null}>
      <About />
    </Suspense>
  )
}
```

Use `fallback={null}` (not a loading spinner) because:
1. Pre-rendered HTML already contains the full page content — the user sees it immediately
2. During hydration, React replaces the pre-rendered content with the hydrated version
3. A spinner would flash briefly and look worse than letting pre-rendered content stay visible
4. `Suspense` with `null` fallback means: "render nothing extra while loading" which is correct for SSR/pre-rendered content

**Apply this pattern to these 12 wrapper files:**
1. `src/routes/home.tsx` — `lazy(() => import('../pages/Home'))`
2. `src/routes/about.tsx` — `lazy(() => import('../pages/About'))`
3. `src/routes/contact.tsx` — `lazy(() => import('../pages/Contact'))`
