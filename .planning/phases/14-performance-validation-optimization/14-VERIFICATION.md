## Gaps Summary

Two truths failed — both stem from the same architectural root cause, unchanged after all planned optimizations.

Mobile Lighthouse Performance (70-74) and mobile LCP (5.04-6.36s) both fail their PERF-05 targets. All planned low-effort optimizations have been applied:

**Optimizations applied (Plans 14-01 through 14-05):**
1. Responsive hero images — 768w WebP variant (84KB vs 390KB original, 78% reduction)
2. Async Google Fonts loading — eliminated render-blocking font stylesheet
3. Hero image preload with responsive imageSrcSet — browser preloads correct viewport variant
4. Self-hosted woff2 fonts — eliminated 2 external DNS/TCP/TLS round-trips to fonts.googleapis.com and fonts.gstatic.com
5. Framer-motion removal — ~67KB uncompressed (~25KB gzipped) JS bundle reduction

**Combined impact on mobile:** +0 to +4 points Performance improvement. LCP essentially unchanged (within measurement variability). The remaining gap is structural.

**Remaining bottlenecks (not addressed by any Phase 14 plan):**
1. **Hero image download** — 84KB WebP over simulated 3G (~200KB/s) takes ~400ms; AVIF format could reduce to ~50KB
2. **React + React Router hydration** — ~316KB JS (entry.client 190KB + React Router 126KB) parsed on 4x-throttled CPU; dominates TBT and blocks LCP paint
3. **woff2 font files not preloaded** — self-hosted fonts declared via CSS @font-face but not `<link rel=preload>`; browser discovers them only after CSS parses

**To reach mobile 90+ would require:**
- `<link rel=preload as=font>` for the 2-3 most-used font weights in `<head>` (easy win, ~200ms LCP)
- Route-level JS code splitting to lazy-load non-critical page chunks (significant TBT reduction)
- AVIF hero image conversion (optional but meaningful)
- Or: explicit product decision to accept PERF-05 mobile criterion as permanently waived

**PERF-05 gap assessment:** The requirement as written in v1.1-REQUIREMENTS.md requires both desktop AND mobile >= 90. Desktop is fully satisfied. Mobile fails across all pages with all planned optimizations applied. This is not a silent gap — it is a documented, measured, acknowledged gap. Further closure requires architectural changes beyond Phase 14 scope, or an explicit product-level decision to waive the mobile criterion.
