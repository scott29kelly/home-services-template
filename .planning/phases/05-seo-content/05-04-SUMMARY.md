## Decisions Made
- Used custom Vite plugin for sitemap generation instead of `vite-plugin-sitemap` (known Vite 7 compatibility issue #88 produces empty sitemaps; custom plugin consistent with existing `copyAndOptimizeImages` pattern)
- Hardcoded hostname `https://example.com` in vite.config.ts with inline comment — cannot ESM-import app config in Vite Node context; user updates this to match `company.url`
- Multiple separate `<JsonLd>` blocks per page rather than merging into one object — Google explicitly supports multiple JSON-LD blocks; easier to conditionally render FAQ and areaServed schemas

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Strip comments from TypeScript source before slug regex extraction**
- **Found during:** Task 2 (sitemap route discovery)
- **Issue:** `service-areas.ts` contains a JSDoc comment `/** URL slug: 'anytown' → /service-areas/anytown */` on the interface field; the regex `/slug:\s*'([^']+)'/g` matched the quoted value inside the comment, producing a duplicate `/service-areas/anytown` URL in the sitemap
- **Fix:** Strip block comments (`/* ... */`) and line comments (`// ...`) from file content before running slug extraction regex
- **Files modified:** `vite.config.ts`
- **Verification:** Sitemap URL count dropped from 55 to 54; `grep -c service-areas/anytown dist/sitemap.xml` returns 1
- **Committed in:** `333058b` (Task 2 commit)
