    2. Update `vite.config.ts`:
       - Replace the inline `readFeatureFlags()`, `getRouteList()` functions with imports from `./src/lib/build-routes`
       - The `generateSitemap()` plugin should call `getAllRoutes()` from the shared module instead of the local `getRouteList()`
       - Keep all other plugins and functions unchanged

    3. Create `react-router.config.ts` in the project root:
       ```typescript
       import { vercelPreset } from '@vercel/react-router/vite'
       import type { Config } from '@react-router/dev/config'
       import { readFeatureFlags, getServiceSlugs, getCitySlugs, getBlogSlugs, getPortfolioSlugs } from './src/lib/build-routes'

       export default {
         appDirectory: 'src',
         buildDirectory: 'build',
         ssr: false,
         presets: [vercelPreset()],
         async prerender({ getStaticPaths }) {
           const flags = readFeatureFlags()

           // Static paths auto-discovered from routes.ts
           const staticPaths = getStaticPaths()

           // Dynamic: service pages (roofing, siding, storm-damage)
           const servicePaths = getServiceSlugs().map(s => `/${s}`)

           // Dynamic: city pages (feature-flag guarded)
           const cityPaths = flags.cityPages
             ? getCitySlugs().map(s => `/service-areas/${s}`)
             : []

           // Dynamic: blog posts (feature-flag guarded)
           const blogPaths = flags.blog
             ? getBlogSlugs().map(s => `/resources/${s}`)
             : []

           // Dynamic: portfolio detail pages
           const portfolioPaths = getPortfolioSlugs().map(s => `/portfolio/${s}`)

           return [
             ...staticPaths,
             ...servicePaths,
             ...cityPaths,
             ...blogPaths,
             ...portfolioPaths,
           ]
         },
       } satisfies Config
       ```

       **IMPORTANT NOTES:**
       - `getStaticPaths()` returns paths for routes that have NO dynamic segments (/, /services, /projects, etc.). It does NOT return paths for routes with `:slug` params â€” those must be explicitly listed.
       - The explicit service routes (`route('roofing', ...)`) in routes.ts ARE static paths â€” `getStaticPaths()` will include them. So `servicePaths` may produce duplicates. That's fine â€” React Router deduplicates pre-render paths.
       - `ssr: false` means no Node.js server runtime needed. The build outputs static HTML + a `__spa-fallback.html` for any non-pre-rendered paths.
       - The `vercelPreset()` configures Vercel to properly serve the static output from CDN.
