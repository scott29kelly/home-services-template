**1. Create `src/hooks/useChatEnhancements.ts`**

This custom hook extracts all chat enhancement logic so both AvaWidget and Ava page share identical behavior. It uses `useLocation()` from react-router-dom for page context.

**Hook signature:**
```typescript
interface UseChatEnhancementsReturn {
  messages: Message[]
  input: string
  setInput: (value: string) => void
  isLoading: boolean
  rateLimited: boolean
  quickActions: string[]
  handleSend: (text?: string) => Promise<void>
  handleKeyDown: (e: React.KeyboardEvent) => void
}

function useChatEnhancements(initialGreeting: string): UseChatEnhancementsReturn
```

**Hook internals:**

a. **Page context** (AVA-01):
   - Call `useLocation()` to get `pathname`
   - Call `getPageContext(pathname)` from `src/lib/chat-context.ts` to get page-specific `quickActions`
   - Call `buildSystemPrompt(pathname)` to generate the dynamic system prompt
   - These are recalculated on every render when pathname changes — context updates on navigation (per CONTEXT.md locked decision)

b. **Message state:**
   - `useState<Message[]>` initialized with `[{ role: 'assistant', content: initialGreeting }]`
   - This is the full UI history — never truncated (API capping happens in `sendMessage()`)

c. **Rate limiting** (AVA-03):
   - `useRef<number>(0)` for `lastSendTimeRef` — tracks last send timestamp
   - `useState<boolean>(false)` for `rateLimited` — controls send button disable + "Please wait..." hint
   - `RATE_LIMIT_MS = 2000` constant
   - In `handleSend()`: check `Date.now() - lastSendTimeRef.current < RATE_LIMIT_MS`. If throttled, set `rateLimited = true`, use `setTimeout` to clear it after the remaining wait time, and return early. Otherwise, update `lastSendTimeRef.current = now`.

d. **Lead capture state machine** (AVA-02):
   - `useRef<LeadState>('idle')` where `type LeadState = 'idle' | 'offered' | 'capturing-name' | 'capturing-phone' | 'capturing-email' | 'captured' | 'declined-once' | 'declined-final'`
   - `useRef<number>(0)` for `exchangeCountRef` — counts user messages only
   - `useRef<{name?: string; phone?: string; email?: string}>({})` for partial lead data
   - The AI handles the natural transition to lead capture via the system prompt instructions — the hook does NOT programmatically inject lead capture messages
   - The hook tracks exchanges and provides the exchange count to the system prompt context so the AI knows when to transition
   - When the AI's response contains a lead capture attempt (detected by checking if `leadStateRef.current === 'idle'` and `exchangeCountRef.current >= 2`), advance state to `'offered'`
   - IMPORTANT: The AI itself drives the conversational flow. The hook only needs to:
     1. Track the exchange count for awareness
     2. Detect when the user provides name/phone/email data in their responses (pattern matching)
     3. Submit the lead via `submitForm()` when enough data is captured
   - When lead data has at minimum name + phone, call `submitForm({ source: 'ava-chat', name, phone, email, page: pathname })` from `src/lib/form-handler.ts`
   - Per CONTEXT.md: name and phone required, email optional. If user provides name and phone, that's enough to submit
   - On lead submission success: advance state to `'captured'`

e. **"Talk to a real person" handling:**
   - When `handleSend()` receives the text `'Talk to a real person'` (exact match), instead of sending to API, inject a local assistant message into the messages array: `"Of course! You can reach our team directly at {company.phone}. {company.phone link formatted as clickable}. I'm also still here if you have any other questions!"`
   - Import `company` from `src/config/company.ts` for the phone number
   - The message content should include the phone number prominently. Since chat renders as text (not HTML), format it simply as: `"Of course! You can reach our team directly at {company.phone}. They're available {company.hours.weekday}. I'm still here if you have more questions!""`
   - Do NOT send this to the API — it's a local injection

f. **handleSend() flow:**
   1. Trim input, return if empty or loading
   2. Rate limit check (return if throttled)
   3. Update `lastSendTimeRef.current`
   4. Increment `exchangeCountRef` (it's a user message)
   5. If text is 'Talk to a real person', inject local response and return
   6. Add user message to messages state
   7. Set loading true
   8. Call `sendMessage(newMessages, systemPrompt)` — systemPrompt from `buildSystemPrompt(pathname)`
   9. Add assistant response to messages
   10. Set loading false

g. **handleKeyDown():**
   - Enter (no shift): prevent default, call handleSend()
   - Escape: does NOT close — that's the component's responsibility (hook doesn't know about open state)
