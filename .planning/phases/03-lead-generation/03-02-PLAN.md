  <action>
    **1. Build `src/components/ui/BookingCalendar.tsx`** -- a custom accessible date picker:

    **Props:** `onSelect: (date: string) => void` (ISO date string yyyy-mm-dd), `selectedDate: string | null`, `blockedDays?: number[]` (day-of-week numbers, default from forms.booking.blockedDays), `maxDaysOut?: number` (default from forms.booking.maxDaysOut)

    **Calendar grid structure:**
    - Header row: month/year display with prev/next month buttons (ChevronLeft, ChevronRight from lucide-react)
    - Day-of-week header: Su Mo Tu We Th Fr Sa
    - Grid of day cells arranged in weeks (rows)
    - Uses `role="grid"` on the table/container, `role="row"` on each week, `role="gridcell"` on each day

    **Day cell rendering:**
    - Each cell has `aria-label` with full date (e.g., "Monday, March 15, 2026")
    - `aria-selected="true"` on selected date
    - `aria-disabled="true"` on past dates and blocked days (per blockedDays config)
    - Visual styling: selected date gets `bg-brand-blue text-white`, today gets `ring-1 ring-brand-blue`, disabled dates get `text-gray-300 cursor-not-allowed`, hover state on available dates
    - Click handler calls `onSelect(isoDateString)` for available dates, no-op for disabled

    **Keyboard navigation (roving tabIndex pattern per ARIA APG):**
    - Only the currently focused date has `tabIndex={0}`. All others have `tabIndex={-1}`.
    - Arrow keys: Left/Right move one day, Up/Down move one week, Home/End move to start/end of week
    - Enter/Space: select the focused date
    - Tab: moves focus OUT of the calendar to the next form element (NOT to the next day)
    - When navigating past month boundary (e.g., right arrow on last day), advance to next month
    - Skip disabled dates when navigating with arrow keys

    **Month navigation:**
    - Prev/next month buttons with aria-labels ("Previous month", "Next month")
    - Cannot navigate to months entirely in the past
    - Cannot navigate beyond maxDaysOut from today

    **Styling:** Match the site's design system -- rounded-xl container, bg-surface, border-border, brand-blue for selection. Compact grid that works well on mobile.

    **2. Build `src/components/forms/BookingForm.tsx`** -- booking form combining calendar + time + contact fields:

    - Import `useForm` from react-hook-form, `zodResolver` from resolvers, `bookingSchema`/`BookingFormData` from schemas
    - Configure with `resolver: zodResolver(bookingSchema)`, `mode: 'onTouched'`
    - Layout:
      1. BookingCalendar component (takes `onSelect` that calls `setValue('preferredDate', date)` and `selectedDate` from `watch('preferredDate')`)
      2. Time range radio group: Morning (8am-12pm), Afternoon (12pm-4pm), Evening (4pm-7pm) -- from forms.booking.timeRanges config. Use `register('preferredTime')` on radio inputs. Style as button-like radio cards.
      3. Contact fields: firstName, lastName, email, phone (same validation as contact form), service dropdown, notes textarea
      4. Honeypot field (same pattern as ContactForm)
      5. Submit button with loading state
    - On submit: call `submitForm(data)`, navigate to '/thank-you' on success
    - Error display: same aria-invalid + role="alert" pattern as ContactForm
    - The BookingCalendar is a visual date picker that sets a hidden field value via RHF's `setValue`. Show validation error for preferredDate below the calendar if user tries to submit without selecting a date.
