## PERF-05 Gap Analysis

### What Passes (5/7 criteria)

| Criterion | Target | Actual | Status |
|-----------|--------|--------|--------|
| Desktop Performance | >= 90 | 91-99 | **PASS** |
| Desktop LCP | < 2.5s | 0.87-1.56s | **PASS** |
| CLS | < 0.1 | 0.000 | **PASS** |
| No render-blocking | Zero external font requests | Self-hosted woff2 | **PASS** |
| Above-fold in HTML | Pre-rendered | Full page HTML | **PASS** |

### What Fails (2/7 criteria)

| Criterion | Target | Actual | Status |
|-----------|--------|--------|--------|
| Mobile Performance | >= 90 | 76-84 | **FAIL** |
| Mobile LCP | < 4.0s | 3.74-4.87s (3/5 pages pass) | **PARTIAL** |

### Root Cause

React hydration on Lighthouse's 4x-throttled mobile CPU simulation. The ~316KB JS bundle (React Router framework entry + page chunks) must parse and execute before hydration completes. This is an architectural bottleneck — not resolvable with further asset optimization.

### Optimizations Exhausted (8 Plans)

1. Responsive hero images (768w/1280w WebP)
2. Hero preload with responsive imageSrcSet
3. Self-hosted woff2 fonts (eliminated 2 external DNS/TCP round-trips)
4. framer-motion removal from shared components (CSS + IntersectionObserver)
5. framer-motion removal from 11 pages + uninstall (~67KB JS reduction)
6. Post-optimization Lighthouse re-measurement
7. woff2 font preloads + AVIF hero image variants
8. React.lazy() code splitting for 12 route wrappers

### Improvement Achieved

| Page | Mobile Perf (baseline) | Mobile Perf (final) | Delta |
|------|----------------------|---------------------|-------|
| Homepage | 70 | 82 | **+12** |
| Roofing | 74 | 84 | **+10** |
| About | 68 | 76 | **+8** |
| City Page | 74 | 84 | **+10** |
| Blog Post | 71 | 81 | **+10** |

### Decision

Accepted as production-ready per STATE.md. Real-world mobile performance exceeds Lighthouse's simulated conditions (4x CPU throttle + simulated 3G). Further work requires SSR or non-React architecture — outside this milestone's scope.
