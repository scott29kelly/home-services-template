Replace framer-motion with CSS animations in all shared components, UI elements, and the useScrollReveal hook. This eliminates ~35-65KB of JS from the hydration bundle, directly reducing mobile TBT.

Purpose: Framer-motion is the largest non-React JS dependency. It adds significant parse+execute time during hydration, particularly impactful under Lighthouse's 4x CPU throttle for mobile. Replacing it eliminates the JS animation runtime entirely.

**Part A: Rewrite src/hooks/useScrollReveal.ts to use native IntersectionObserver**

Replace the entire file. The new hook returns `{ ref, isInView }` with the same API signature so callers don't need changes to their conditional class logic:

```typescript
import { useRef, useState, useEffect } from 'react'

export function useScrollReveal(margin = '-100px 0px') {
  const ref = useRef<HTMLElement | null>(null)
  const [isInView, setIsInView] = useState(false)

  useEffect(() => {
    const el = ref.current
    if (!el) return

    // Respect prefers-reduced-motion: skip animation, show immediately
    if (window.matchMedia('(prefers-reduced-motion: reduce)').matches) {
      setIsInView(true)
      return
    }

    const observer = new IntersectionObserver(
      ([entry]) => {
        if (entry.isIntersecting) {
          setIsInView(true)
          observer.disconnect() // once: true equivalent
        }
      },
      { rootMargin: margin }
    )

    observer.observe(el)
    return () => observer.disconnect()
  }, [margin])

  return { ref, isInView }
}
```

This preserves the exact same return type `{ ref, isInView }` so all consuming components continue to work. The `once: true` behavior is replicated by disconnecting the observer after first intersection.

The `prefers-reduced-motion` check replaces framer-motion's `MotionConfig reducedMotion="user"` — elements immediately appear without animation for users who prefer reduced motion.
