  <name>Task 1: Convert Google Fonts to async loading and fix hero preload</name>
  <files>src/root.tsx</files>
  <action>
Make two changes to `src/root.tsx` to eliminate render-blocking resources and fix the hero preload.

**Part A: Async Google Fonts loading**

Replace the current render-blocking Google Fonts `<link rel="stylesheet">`:
```tsx
<link
  href="https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700&family=Plus+Jakarta+Sans:wght@600;700;800&display=swap"
  rel="stylesheet"
/>
```

With the async preload + onload pattern:
```tsx
{/* Google Fonts — async loading eliminates render-blocking resource */}
<link rel="preconnect" href="https://fonts.googleapis.com" />
<link rel="preconnect" href="https://fonts.gstatic.com" crossOrigin="" />
<link
  rel="preload"
  as="style"
  href="https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700&family=Plus+Jakarta+Sans:wght@600;700;800&display=swap"
  onLoad={(e: React.SyntheticEvent<HTMLLinkElement>) => {
    e.currentTarget.rel = 'stylesheet'
  }}
/>
<noscript>
  <link
    rel="stylesheet"
    href="https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700&family=Plus+Jakarta+Sans:wght@600;700;800&display=swap"
  />
</noscript>
```

Note: The two `<link rel="preconnect">` lines already exist in root.tsx. Keep them where they are (do not duplicate). Only replace the `<link ... rel="stylesheet">` line with the `<link rel="preload" as="style" ...>` and `<noscript>` block.

The user decision accepts minor FOUT (flash of unstyled text). The `display=swap` parameter in the URL handles this at the CSS level.

**Part B: Update hero image preload to use imagesrcset**

Replace the current hero preload:
```tsx
<link rel="preload" as="image" href="/images/hero-roofing.webp" type="image/webp" />
```

With a responsive-aware preload:
```tsx
<link
  rel="preload"
  as="image"
  href="/images/hero-roofing.webp"
  imageSrcSet="/images/hero-roofing-768w.webp 768w, /images/hero-roofing-1280w.webp 1280w, /images/hero-roofing.webp 1920w"
  imageSizes="100vw"
  type="image/webp"
/>
```

Note: React uses camelCase `imageSrcSet` and `imageSizes` for the HTML `imagesrcset` and `imagesizes` attributes. This ensures the browser preloads the correctly-sized hero variant for the viewport width, not always the full 1920px image.

This also addresses the duplicate preload issue noted in the research: by having a single explicit preload with `imageSrcSet`, the browser uses this instead of generating a second one. If React Router's `Links` component was generating a duplicate, the explicit preload with `imageSrcSet` takes precedence and the browser deduplicates matching preloads.
