<objective>
Self-host Inter and Plus Jakarta Sans font files to eliminate Google Fonts external origin DNS/TCP round-trips that cause 1-2s LCP penalty on Lighthouse mobile (simulated 3G).

Purpose: The verification gaps show mobile LCP at 4.97-6.72s (target 4.0s) and mobile Performance at 68-74 (target 90+). Google Fonts requires DNS lookup + TCP connection + TLS handshake to two external origins (fonts.googleapis.com and fonts.gstatic.com) on every page load. On Lighthouse's simulated slow 3G, each round-trip adds ~300-600ms. Self-hosting eliminates these external round-trips entirely — fonts are served from the same origin as the HTML.

Output: Self-hosted woff2 font files in src/fonts/, @font-face declarations in index.css, Google Fonts links removed from root.tsx.
</objective>

Download woff2 font files for Inter and Plus Jakarta Sans from Google Fonts (or google-webfonts-helper / fontsource).

The project uses these specific weights:
- **Inter:** 400 (regular), 500 (medium), 600 (semibold), 700 (bold) — body text
- **Plus Jakarta Sans:** 600, 700, 800 (extrabold) — headings

Steps:

1. Create directory `src/fonts/`

2. Download woff2 files for each weight. Use the google-webfonts-helper API or download directly from Google Fonts CSS. The approach:

   ```bash
   # Fetch the Google Fonts CSS with woff2 user-agent to get woff2 URLs
   curl -H "User-Agent: Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36" \
     "https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700&family=Plus+Jakarta+Sans:wght@600;700;800&display=swap" \
     -o /tmp/google-fonts.css
   ```

   Parse the CSS to extract woff2 URLs, then download each file:
   ```bash
   # Extract woff2 URLs and download
   grep -oP "url\(\K[^)]+\.woff2" /tmp/google-fonts.css | while read url; do
     filename=$(echo "$url" | grep -oP "[^/]+$")
     curl -L "$url" -o "src/fonts/$filename"
   done
   ```

   Alternatively, use `fontsource` npm packages:
   ```bash
   # If curl approach fails, install fontsource packages temporarily
   npm install --save-dev @fontsource/inter @fontsource/plus-jakarta-sans
   # Then copy the woff2 files from node_modules to src/fonts/
   # and uninstall the packages afterward
   ```

   The simplest reliable approach: use Node.js to fetch the CSS and download fonts:
   ```javascript
   // Download script (run once, then delete)
   const css = await fetch('https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700&family=Plus+Jakarta+Sans:wght@600;700;800&display=swap', {
     headers: { 'User-Agent': 'Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36' }
   }).then(r => r.text());

   // Parse woff2 URLs and download each
   const urls = [...css.matchAll(/url\(([^)]+\.woff2)\)/g)].map(m => m[1]);
   // Download each URL to src/fonts/ with a meaningful name
   ```

3. Name files descriptively:

   - `inter-v18-latin-500.woff2`

**IMPORTANT:** Only download latin subset woff2 files. This project serves English content only. Do not download latin-ext, cyrillic, greek, or vietnamese subsets — they add unnecessary bytes.

**Part B: Remove ALL Google Fonts links from src/root.tsx**

Remove the following lines entirely from the `<head>` section:

1. The preconnect links:

2. The async preload link (and its comment):
```tsx
{/* Google Fonts — async loading eliminates render-blocking resource */}
<link
  rel="preload"
  as="style"
  href="https://fonts.googleapis.com/css2?family=Inter:wght@400;500;600;700&family=Plus+Jakarta+Sans:wght@600;700;800&display=swap"
  onLoad={(e: React.SyntheticEvent<HTMLLinkElement>) => {
    e.currentTarget.rel = 'stylesheet'
  }}
/>
```

3. The noscript fallback:
```tsx
<noscript>
```
