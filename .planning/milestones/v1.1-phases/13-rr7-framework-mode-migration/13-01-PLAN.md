       - Include `<Meta />` and `<Links />` from `react-router` in `<head>` (enables per-route meta)

       - Include `<ScrollRestoration />` and `<Scripts />` from `react-router` before `</body>`
       - Export a default `Root` function that renders `<Outlet />`
       - Import `./index.css` at the top

    2. Create `src/entry.client.tsx` — browser hydration entry:
       - Import `{ StrictMode }` from `react`
       - Import `{ hydrateRoot }` from `react-dom/client`
       - Import `{ HydratedRouter }` from `react-router/dom`
       - Call `hydrateRoot(document, <StrictMode><HydratedRouter /></StrictMode>)`
       - Do NOT import reportWebVitals — that is dead code being removed in Phase 15

    3. Create `src/entry.server.tsx` — server/pre-render entry:
       - Import `{ PassThrough }` from `node:stream`
       - Import `type { EntryContext }` from `react-router`
       - Import `{ createReadableStreamFromReadable }` from `@react-router/node`
       - Import `{ ServerRouter }` from `react-router`
       - Import `{ renderToPipeableStream }` from `react-dom/server`
       - Export default `handleRequest(request, responseStatusCode, responseHeaders, routerContext)` function
       - Use the streaming pattern from the research (renderToPipeableStream with onShellReady/onShellError)
       - Set 5 second abort timeout

    4. Create `src/routes.ts` — route configuration:
       - Import `{ type RouteConfig, index, route, layout }` from `@react-router/dev/routes`
       - Use the layout wrapper pattern: `layout('routes/layout.tsx', [...])`
       - Define ALL routes using explicit paths (not dynamic `:serviceSlug`):
         - `index('routes/home.tsx')` for Home
         - `route('roofing', 'routes/service-page.tsx')` for each service (roofing, siding, storm-damage) — explicit routes avoid collision with other top-level routes
         - `route('services', 'routes/services.tsx')`
         - `route('projects', 'routes/projects.tsx')`
         - `route('portfolio/:slug', 'routes/portfolio.$slug.tsx')`
         - `route('testimonials', 'routes/testimonials.tsx')`
         - `route('about', 'routes/about.tsx')`
         - `route('contact', 'routes/contact.tsx')`
         - `route('thank-you', 'routes/thank-you.tsx')`
         - `route('service-areas', 'routes/service-areas.tsx')`
         - `route('service-areas/:slug', 'routes/service-areas.$slug.tsx')`
         - `route('ava', 'routes/ava.tsx')`
         - `route('financing', 'routes/financing.tsx')`
         - `route('resources', 'routes/resources.tsx')`
         - `route('resources/:slug', 'routes/resources.$slug.tsx')`
         - `route('*', 'routes/$.tsx')` for 404 catch-all
       - All routes are always declared (feature flags are handled at the route module level in loaders/components, NOT in routes.ts)

    5. Create `src/routes/layout.tsx` — the shared layout route:
       - This is a NEW file wrapping the existing Layout component functionality
       - Import `{ Outlet, useLocation }` from `react-router` (NOT react-router-dom)
       - Import `{ SpeedInsights }` from `@vercel/speed-insights/react`
       - Import Header, Footer, ScrollToTop, AvaWidget, StickyMobileCTA, AnnouncementBanner from existing component locations (use relative paths: `../components/layout/Header`, etc.)
       - Import `{ features }` from `../config`
       - Reproduce the same layout structure as `src/components/layout/Layout.tsx`:
         - ScrollToTop, skip-to-content link, AnnouncementBanner, Header, main with Outlet, Footer, StickyMobileCTA, conditional AvaWidget
       - Add SpeedInsights conditionally: `{import.meta.env.VITE_VERCEL_SPEED_INSIGHTS === 'true' && <SpeedInsights />}`
       - Export as default function `AppLayout()`

    6. Do NOT delete `index.html`, `main.tsx`, `App.tsx`, or `Layout.tsx` yet — they will be cleaned up after route modules are working. The framework mode ignores `index.html` (reactRouter plugin takes over).
