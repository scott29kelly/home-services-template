## Partial Requirement Details

### CFG-05: 404 Page (MUST) — 3/4 criteria

**Met:** Styled 404 page inside Layout, navigation links to key pages, noindex meta tag.
**Missing:** Phone CTA (tel: link) on 404 page.
**Impact:** Low — users on 404 pages cannot directly call; they must navigate away first.
**Fix effort:** ~5 minutes — add a `tel:` link importing `company.phone` from config.

### CFG-06: Feature Flags System (MUST) — Integration gap

**Met:** All 6 flags exist (assistant, blog, financingCalculator, onlineBooking, cityPages, beforeAfter). Flags correctly gate nav links, routes, footer links, sitemap entries, and UI sections.
**Missing:** AvaWidget floating button in Layout.tsx not gated by `features.assistant`.
**Impact:** Medium — disabling the assistant feature still shows the chat widget on every page.
**Fix effort:** ~2 minutes — add `features.assistant &&` to Layout.tsx line 31.

### SEO-04: Sitemap & Robots.txt (MUST) — Integration gap

**Met:** Custom Vite plugin generates sitemap.xml with 54 URLs. Feature flags gate routes. City pages, blog posts, and portfolio slugs auto-derived from config.
**Missing:** Service routes hardcoded as `['/roofing', '/siding', '/storm-damage']` instead of derived from services.ts. Hostname hardcoded as 'https://example.com'.
**Impact:** Medium — adding a new service to config won't auto-add it to sitemap. Template users must update vite.config.ts hostname.
**Fix effort:** ~15 minutes — parse service slugs from services.ts (same pattern used for cities), read hostname from company.ts.

### PERF-02: Performance Monitoring (COULD) — Lighthouse target not met

**Met:** web-vitals reports CLS/INP/LCP. Vercel Speed Insights opt-in. Accessibility 90, SEO 100.
**Missing:** Lighthouse Performance 87 desktop / 51 mobile (target 90+).
**Impact:** Low — COULD priority, not a launch blocker. Root cause is SPA architecture (no SSR).
**Fix effort:** Significant — requires SSR/pre-rendering migration. Deferred to future milestone.
